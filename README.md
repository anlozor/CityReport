# CityReport
Este trabajo propone el diseño e implementación de una aplicación web destinada a la gestión y visualización de incidencias ciudadanas. El objetivo es proporcionar una herramienta que permita a los “usuarios” reportar problemas urbanos (como calles cortadas por obras o cualquier otro motivo, semáforos averiados, desperfectos en mobiliario urbano, etc), así como a los “gestores de incidencias” poder ver, actualizar e informar al respecto de cada incidencia. La idea es que la aplicación incluya un mapa interactivo para ver las incidencias y generarlas. Además de incluir filtros como “fecha”, “incidencia completada” o “incidencia en curso”, o “tipo de incidencia” (calle cortada, desperfecto en asfalto, objeto obstruyendo, etc.).

Lista de objetivos concretos: 
1.	Revisar el estado del arte de aplicaciones de gestión de incidencias ciudadanas: Investigar qué sistemas o aplicaciones similares existen, cómo funcionan, problemas que resuelven, y limitaciones que tienen.
2.	Definir requisitos funcionales y técnicos: Decidir qué debe hacer la aplicación (funcionalidades) y cómo se va a construir (tecnologías, plataformas…).
3.	Diseñar la arquitectura del software y el modelo de datos: Planificar cómo se va a organizar el software y cómo se van a almacenar los datos.
4.	Implementar la aplicación web, incluyendo registro de incidencias y visualización en un mapa.
5.	Desarrollar el panel de control para clasificación, filtrado y análisis básico de las incidencias: Crear un área para “gestores” donde puedan organizar las incidencias y obtener información útil.
6.	Realizar pruebas.
7.	Documentar todo el proceso de desarrollo, incluyendo arquitectura, código y pruebas.
